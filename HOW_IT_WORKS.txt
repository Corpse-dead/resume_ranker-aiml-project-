📖 OVERVIEW

This system automatically ranks candidate resumes based on how
well they match a given job description. It uses Natural Language
Processing (NLP) and Machine Learning techniques to analyze text
and calculate similarity scores.

═══════════════════════════════════════════════════════════════

🔄 SYSTEM WORKFLOW

The system follows these steps:

STEP 1: INPUT
├─ Job Description (what you're looking for)
└─ Multiple Resumes (candidates to evaluate)

STEP 2: TEXT PREPROCESSING
├─ Convert all text to lowercase
├─ Remove URLs, emails, special characters
├─ Break text into individual words (tokenization)
├─ Remove common words like "the", "is", "and" (stopwords)
└─ Clean unnecessary symbols

STEP 3: FEATURE EXTRACTION (TF-IDF)
├─ Convert cleaned text into numerical vectors
├─ Calculate term frequency (how often words appear)
├─ Calculate inverse document frequency (how unique words are)
└─ Combine to create TF-IDF scores (important word weights)

STEP 4: SIMILARITY CALCULATION
├─ Compare job description vector with each resume vector
├─ Use Cosine Similarity to measure similarity (0 to 1)
└─ Generate similarity score for each resume

STEP 5: RANKING
├─ Sort all resumes by similarity score (highest first)
├─ Assign ranks (1st, 2nd, 3rd, etc.)
└─ Display results with percentages

STEP 6: OUTPUT
├─ Show ranked list on screen
└─ Save results to CSV file (rankings_output.csv)

═══════════════════════════════════════════════════════════════

🧠 KEY CONCEPTS EXPLAINED

1. TEXT PREPROCESSING
   - Raw text has noise (punctuation, special chars, etc.)
   - We clean it to make analysis accurate
   - Example: "Python Developer!!!" → "python developer"

2. STOPWORDS
   - Common words that don't add meaning
   - Examples: "the", "is", "at", "and", "or"
   - Removed because they don't help distinguish resumes

3. TF-IDF (Term Frequency - Inverse Document Frequency)
   - Converts words into numbers computers can understand
   - Gives high scores to important, distinctive words
   - Gives low scores to common, less meaningful words
   
   Example:
   - "Python" appears often in a resume → High TF
   - "Python" only in 2 out of 10 resumes → High IDF
   - Result: "Python" gets high TF-IDF score (important!)

4. COSINE SIMILARITY
   - Measures how similar two documents are
   - Range: 0 (completely different) to 1 (identical)
   - Calculates the angle between document vectors
   
   Example:
   - Job wants "Python ML Engineer"
   - Resume A: "Python ML Developer" → High similarity (0.85)
   - Resume B: "Marketing Manager" → Low similarity (0.05)

5. RANKING
   - Sort resumes by similarity score (descending order)
   - Best match gets Rank 1
   - Worst match gets lowest rank

📊 EXAMPLE SCENARIO

JOB DESCRIPTION:
"Looking for a Python Developer with Machine Learning and 
Data Analysis skills."

RESUMES:
Resume 1: "Python Developer with Machine Learning experience"
Resume 2: "Marketing Manager with strong communication skills"
Resume 3: "Data Analyst skilled in Python and statistics"

SYSTEM PROCESSING:

1. Preprocessing:
   Resume 1: ["python", "developer", "machine", "learning", ...]
   Resume 2: ["marketing", "manager", "communication", ...]
   Resume 3: ["data", "analyst", "python", "statistics", ...]

2. TF-IDF Vectorization:
   Converts each into numerical vectors showing word importance

3. Similarity Calculation:
   Job vs Resume 1: 0.8234 (82.34% match)
   Job vs Resume 2: 0.0323 (3.23% match)
   Job vs Resume 3: 0.6521 (65.21% match)

4. Final Ranking:
   Rank 1: Resume 1 - 82.34% (Best Match!)
   Rank 2: Resume 3 - 65.21% (Good Match)
   Rank 3: Resume 2 - 3.23% (Poor Match)

EXPLANATION:
- Resume 1 has "Python", "Machine Learning" → High match
- Resume 3 has "Python", "Data", "Analyst" → Medium match
- Resume 2 has "Marketing" (different field) → Low match

💻 TECHNICAL COMPONENTS

MODULE 1: preprocess.py
- Handles all text cleaning operations
- Removes noise and standardizes text
- Uses NLTK library for tokenization and stopword removal

MODULE 2: vectorizer.py
- Converts text to TF-IDF vectors
- Uses scikit-learn's TfidfVectorizer
- Creates numerical representation of documents

MODULE 3: similarity.py
- Calculates cosine similarity between vectors
- Compares job description with all resumes
- Generates similarity scores (0 to 1)

MODULE 4: ranker.py
- Coordinates all modules together
- Organizes results into ranked list
- Saves output to CSV file

MODULE 5: main.py
- Entry point of the application
- User interface and display
- Handles file loading and error checking

📈 WHY SCORES AREN'T 100%?

You might notice similarity scores like 36.31% or 65.21% rather
than 100%. This is normal and expected because:

1. Different Vocabulary: Even relevant resumes use different words
2. Length Differences: Resumes have varying lengths
3. Unique Content: Each resume has unique experiences
4. Realistic Matching: Perfect 100% would mean identical text

INTERPRETATION GUIDE:
- 60%+  : Excellent match (highly relevant)
- 40-60%: Good match (relevant with some gaps)
- 20-40%: Fair match (some relevant content)
- <20%  : Poor match (different domain/skills)
